### 3-1. Thinking Depth 모드

```
┌───────────────────────────────────────────────────┐
│ ┌──────────────────────────────────────────────┐  │
│ │ [+] [🎤] │ Type a message...                 │  │
│ └──────────────────────────────────────────────┘  │
│                                                    │
│ [⚡ Fast] [⚖️ Balanced] [🧠 Deep]  [Model ▾] [→]  │
│                                                    │
│ ⚡ Fast: 빠른 응답, 간단한 질문에 적합              │
│ ⚖️ Balanced: 기본 모드, 균형 잡힌 응답              │
│ 🧠 Deep: 깊은 추론, 복잡한 문제에 적합              │
└───────────────────────────────────────────────────┘
```

**구현**: PromptInput에 thinking depth 토글 추가. 모델별 매핑:
- Fast → temperature 낮음, max_tokens 제한
- Balanced → 기본 설정
- Deep → Claude `extended_thinking`, GPT `reasoning_effort: high`

## 4. 구현 계획

### Phase 1: Thinking Depth + 예산 경고 (1일)

**1-1. Thinking Depth 모드**

신규/수정 파일:
- `src/shared/types/index.ts` — `ThinkingDepth = 'fast' | 'balanced' | 'deep'`
- `src/entities/settings/settings.store.ts` — `thinkingDepth` 상태
- `src/widgets/prompt-input/PromptInput.tsx` — depth 토글 UI
- `src/shared/lib/providers/factory.ts` — depth → 모델 파라미터 매핑

구현 요점:
```typescript
// factory.ts — depth 매핑
function applyThinkingDepth(config: RequestConfig, depth: ThinkingDepth): RequestConfig {
  switch (depth) {
    case 'fast':
      return { ...config, temperature: 0.3, max_tokens: 1024 }
    case 'deep':
      // Claude: extended_thinking, GPT: reasoning_effort
      return { ...config, extended_thinking: true, max_tokens: 16384 }
    default:
      return config
  }
}
```

## 6. 기술적 의사결정

| 결정 | 선택 | 이유 |
|------|------|------|
| Thinking Depth | 모델 파라미터 매핑 | 별도 모델 불필요, 기존 프로바이더 활용 |
| 가드레일 | 클라이언트 Regex | 서버 불필요, 즉시 감지, 오프라인 동작 |

| 차트 생성 | SVG (기존 BarChart 확장) | 추가 라이브러리 불필요 |
| 문서 검사 | LLM 1회 호출 | 별도 NLP 엔진 불필요, 다국어 지원 |
